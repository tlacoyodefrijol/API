Organizaciones ¶

Recupera datos de una lista de organizaciones.

Endpoint

/api/organizations

/api/organizations?param=value&param2=value2

Parámetros de URL

Puedes agregar cualquier de las Propiedades de Organización como parámetro y la API filtrará por organizaciones con esa propiedad.

Propiedades de respuesta

páginas
Diccionario de enlaces de paginación, incluyendo opcionalmente first, prev, next, y last.
objetos
Lista de organizaciones individuales.

Solicitud muestra

{{ api_base }}/api/organizations

Respuesta muestra

{
  "pages": {
    "next": …,
    "last": …
  },
  "objects": [

     "type": "Brigade",
     "name": "Open Oakland",
     …

     "type": "Code for All",
     "name": "Code for Berlin",
     …

   …
  ]
}

GeoJSON de organizaciones ¶

Obtiene una lista de organizaciones, en formato GeoJSON para aplicaciones geográficas..

Endpoint

/api/organizations.geojson

Propiedades de respuesta

type
GeometryCollection
features
Lista de objetos con estas propiedades:
type
Característica
id
Identificador único para la organización.
geometry
Punto objeto geométrico con la ubicación de la organización.
properties
Propiedades de características, seleccionadas de propiedades para una sola organización.

Solicitud muestra

{{ api_base }}/api/organizations.geojson

        "name": "Open Oakland",
        "website": "http://openoakland.org/",
        "city": "Oakland, CA",
        …
      }
    },
    …
  ]
}

Una Organización ¶

Devuelve una sola organización.

Endpoint

/api/organization/{organization id}

Propiedades de Organización ¶

name
Nombre oficial.
website
Sitio de la organización
city
Ciudad de la organización.
latitude
Latitud geográfica en grados.
longitude
Longitud geográfica en grados.
api_url
Enlace a la información completa del API para esta organización.
started_on
Fecha de inicio opcional para la organización.
current_events
Selección de diccionario de eventos.
all_events
çEnlace a la información completa del API para los eventos de una organización.
events_url
URL fuente de la lista de eventos.
current_projects
The three most recently updated projects of this organization.
all_projects
Link to API information for all organization projects.
projects_list_url
URL source for projects list.
current_stories
Selection of current story dictionaries.
all_stories
Link to API information for all organization stories.
rss
URL source for stories list.

Projects ¶

Retrieve a list of projects. Projects are ordered by most recent update.

Endpoints

/api/projects
/api/projects?param=value&param2=value2
/api/organization/{organization id}/projects

Url parameters

You can add any of the Project properties as a parameter and the API will filter by Organizations that have that property. You can also do request projects based on organization properties such as:

/api/project/organization_type=Brigade
/api/project/organization_city=San%20Francisco,%20CA

Response Properties

pages
Dictionary of pagination links, optionally including first, prev, next, and last.
objects
List of individual projects.

Events ¶

Retrieve a list of events.

Endpoints

/api/events
/api/events/upcoming_events
/api/events/past_events
/api/organization/{organization id}/events
/api/organization/{organization id}/events/upcoming_events
/api/organization/{organization id}/events/past_events

Response Properties

pages
Dictionary of pagination links, optionally including first, prev, next, and last.
objects
List of individual events.
total
The total count of objects.

Event filters

upcoming_events
Only returns events happening in the future.
past_events
Only returns events that have already happened.

Issues ¶

Retrieve a list of project issues.

Endpoints

/api/issues
/api/issues/{issue id}
/api/issues/labels/{labels}

Response Properties

pages
Dictionary of pagination links, optionally including first, prev, next, and last.
objects
List of individual issues.

Issue filters

labels (comma separated)
Return issues that have atleast one of the given labels

Issue parameters

Any issue attribute
id, title, body, etc ...
Any project attribute
name, type, html_url, etc ...
Any organization attribute
name, website, city, etc ...
